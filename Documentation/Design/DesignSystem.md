# 锻炼YA Design System

Version: 1.0


# 1. 设计理念

锻炼YA的体验核心：

鸭子姿态、轻松开始、陪伴。


设计目标：

让用户打开App时，
感受到：

- 温暖
- 安心
- 积极
- 长期成长


锻炼YA不是竞技型健身产品。

不是强调：

- 比别人更强
- 更快达到目标

而强调：

每天照顾身体一点点。



---

# 2. 核心视觉符号


## Duck Home 鸭子姿态

鸭子是首页课程入口的核心交互主角。

Visual Reference：

`/Users/dave/Desktop/745A572E-2A49-4163-942D-708426833444.png`

参考图用于记录当前 Duck Home 首页视觉基准。
它不是立即实现 UI 的要求。

角色原则：

- 首页核心角色是鸭子 Duck Mascot
- 鸭子不是装饰元素
- 鸭子是用户进入训练体验的交互入口
- 用户通过鸭子的姿态变化理解 Standing / Sitting / Lying 三种身体照顾状态

默认状态：

- 站立，对应 Standing Care

向左滑动：

- 坐在椅子上，对应 Sitting Care

向右滑动：

- 躺在床上，对应 Lying Care

姿态对应：

- Standing：适合站姿 Care Routine
- Sitting：适合坐姿 Care Routine
- Lying：适合躺姿 Care Routine

设计目的：

让用户先通过姿态理解“我现在应该用什么姿势照顾身体”，
而不是先理解运动目标或身体部位知识。

视觉方向：

- 温暖
- 轻松
- 陪伴感
- 低压力
- 面向全年龄用户
- 不强调健身竞技
- 强调日常身体照顾

# 3. 色彩系统


整体：

自然色系。


关键词：

- 春天
- 夏天
- 秋天
- 冬天


颜色根据季节变化。

避免：

高刺激健身App风格。


不使用：

强烈警告红色

过度竞技色彩



---

# 4. 动画原则


动画目标：

鼓励，而不是评价。


训练完成后：

播放2-3秒动画。


例如：

腰背训练完成：

1. 卡通人物弯腰
2. 腰部出现不适提示
3. 提示逐渐消失
4. 身体恢复挺直
5. 人物微笑并点赞


动画表达：

“你今天照顾了自己的身体。”


不是：

“你的疾病被治愈。”



---

# 5. UI风格


关键词：

- 简洁
- 柔和
- 有生命感
- 陪伴感


避免：

传统健身App：

- 大数字
- 卡路里竞争
- 排名
- 强目标压力

- 圆角
- 呼吸感
- 轻量动画

PostureInteractionView

WorkoutCard

AnimationView
